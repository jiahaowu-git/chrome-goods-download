  <name>Task 2: Add error tracking and alert with warning count</name>

  <action>
Modify the existing downloadImages event listener to track failed images and show an alert on completion with the count of failed images.

**Implementation approach:**

Inside the for loop that processes batches, track failures:
```javascript
let failedCount = 0;

for (let i = 0; i < checkedImages.length; i += batchSize) {
  const batch = checkedImages.slice(i, i + batchSize);
  try {
    const jpgBlobs = await Promise.all(
      batch.map(img => fetchImageAsJpgBlob(img.url))
    );
    jpgBlobs.forEach((blob, idx) => {
      const fileName = `${String(i + idx + 1).padStart(3, '0')}.jpg`;
      zip.file(fileName, blob);
    });
    processed += jpgBlobs.length;
  } catch (err) {
    console.warn('Image processing failed:', err);
    failedCount += batch.length;
    // Continue with next batch
  }
}
```

After the zip is generated and saveAs is called, add:
```javascript
if (failedCount > 0) {
  alert(`${failedCount} 张图片下载失败，已跳过`);
}
```

Per 04-RESEARCH.md: "Skip failed images with warning count. Show toast/alert with 'N 张图片下载失败' on completion."
