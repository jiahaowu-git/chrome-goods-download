**fetchImageAsJpgBlob(url):**
- fetch(url) returns Response with ArrayBuffer
- Create Blob from ArrayBuffer: new Blob([arrayBuffer])
- Create object URL: URL.createObjectURL(blob)
- Load into temporary Image element
- On image load: create canvas, draw image at natural dimensions, call canvas.toBlob with 'image/jpeg' quality 0.92
- Revoke object URL in both onload and onerror paths
- Return Promise resolving to JPG Blob

**sanitizeFilename(name):**
- Replace illegal filename characters (/, \, :, *, ?, ", &lt;, &gt;, |) with underscore using regex: /[\\/:*?"&lt;&gt;|]/g
- Trim whitespace
- Limit to 200 characters

Key implementation notes (per 04-RESEARCH.md anti-patterns):
- Use URL.createObjectURL from fetched blob (NOT img.src=url directly) to avoid tainted canvas
- Always pair URL.createObjectURL with URL.revokeObjectURL in cleanup paths
- Use async/await pattern for clarity

```javascript
async function fetchImageAsJpgBlob(url) {
  const response = await fetch(url);
  if (!response.ok) throw new Error(`Failed to fetch ${url}: ${response.status}`);
  const arrayBuffer = await response.arrayBuffer();
  const blob = new Blob([arrayBuffer]);
  const objectUrl = URL.createObjectURL(blob);

  return new Promise((resolve, reject) => {
    const img = new Image();
    img.onload = () => {
      try {
        const canvas = document.createElement('canvas');
        canvas.width = img.naturalWidth;
        canvas.height = img.naturalHeight;
        const ctx = canvas.getContext('2d');
        ctx.drawImage(img, 0, 0);
        canvas.toBlob((jpgBlob) => {
          URL.revokeObjectURL(objectUrl);
          if (jpgBlob) resolve(jpgBlob);
          else reject(new Error('Canvas toBlob returned null'));
        }, 'image/jpeg', 0.92);
      } catch (e) {
        URL.revokeObjectURL(objectUrl);
        reject(e);
      }
    };
    img.onerror = () => {
      URL.revokeObjectURL(objectUrl);
      reject(new Error('Failed to load image'));
    };
    img.src = objectUrl;
  });
}

function sanitizeFilename(name) {
  return name.replace(/[\\/:*?"&lt;&gt;|]/g, '_').trim().substring(0, 200);
}
```

**Implementation (per 04-RESEARCH.md complete download flow):**

```javascript
window.addEventListener('downloadImages', async (event) => {
  const { images, pageTitle } = event.detail;
  const checkedImages = images.filter(img => img.checked);

  if (checkedImages.length === 0) return;

  const zip = new JSZip();
  const sanitizedTitle = sanitizeFilename(pageTitle) || 'download';

  // Process images in batches of 5 for memory management
  const batchSize = 5;
  let processed = 0;

  for (let i = 0; i < checkedImages.length; i += batchSize) {
    const batch = checkedImages.slice(i, i + batchSize);
    try {
      const jpgBlobs = await Promise.all(
        batch.map(img => fetchImageAsJpgBlob(img.url))
      );
      jpgBlobs.forEach((blob, idx) => {
        const fileName = `${String(i + idx + 1).padStart(3, '0')}.jpg`;
        zip.file(fileName, blob);
      });
      processed += jpgBlobs.length;
    } catch (err) {
      console.warn('Batch processing error:', err);
      // Continue with next batch even if this one partially failed
    }
  }

  // Generate zip and trigger download
  const zipBlob = await zip.generateAsync({ type: 'blob' });
  saveAs(zipBlob, `${sanitizedTitle}.zip`);
});
```

Key behaviors:
- Filters images to only checked items (DOWN-01)
- Uses extension host_permissions for cross-origin fetch (DOWN-02)
- Converts each image to JPG via canvas at quality 0.92 (DOWN-03)
- Adds all JPGs to JSZip zip (DOWN-04)
- Names zip from pageTitle after sanitization (DOWN-05)
- Uses FileSaver.saveAs to trigger browser download (DOWN-06)
- Batches of 5 images for memory management (per 04-RESEARCH.md)
- Sequential padded filenames (001.jpg, 002.jpg, etc.) per research

Button UX (disable during processing) and error handling (warning count) are handled in Wave 2 Plan 04-02.
